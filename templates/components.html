The Fluid Grid

indigo.css comes equipped with a full set of column classes for different devices. Having separate classes for different devices simplifies responsive web design.

The columns shrink and stretch dynamically with the device/window. When the width of the device/window is less than 550 pixels, each column's width is set to 100%, fitting the full width of the device/window.

Mobile column width is set to 100% by default. For more control on mobile views, a full set of mobile column classes is also available.

Check out the following code example to see how the above grid was created.

  
<div class="row">
  <div class="col-m-12">
    <p>12</p>
  </div>
</div>

  <div class="col-m-1">

</div>

Small Devices (Mobile)

The small device class covers devices that are between 400 and 550 pixels wide.

Mobile columns are set to 100% width by default. If you don't plan on changing that, then you don't need to worry about this class.

However, if you would like greater control over your mobile views, use the base class .col- followed by the desired number of columns. If using more than one column in a single row, the amount of columns must add up to exactly twelve (referencing the example below, 6+6=12).

  
<div class="row">
  <div class="col-6"></div>
  <div class="col-6"></div>
</div>

Note that in order for the small device class to work, you will need to comment out line 35 in grid.css.

  
[class*="col-"] {
  width: 100%; /* Comment out this line to use mobile column classes! */
  float: left;
}

Medium Devices (Phablets & Tablets)

The medium device class covers devices that are between 550 and 1024 pixels wide.

Use the base class .col-m- followed by the desired number of columns (up to 12). If using more than one column in a single row, the amount of columns must add up to exactly twelve (referencing the example below, 4+4+4=12).

  
<div class="row">
  <div class="col-m-4"></div>
  <div class="col-m-4"></div>
  <div class="col-m-4"></div>
</div>

Large Devices (Laptops & Desktops)

The large device class covers devices that are wider than 1024 pixels.

Use the base class .col-l- followed by the desired number of columns (up to 12). If using more than one column in a single row, the amount of columns must add up to exactly twelve (referencing the example below, 3+6+3=12).

  
<div class="row">
  <div class="col-l-3"></div>
  <div class="col-l-6"></div>
  <div class="col-l-3"></div>
</div>
MORE GRID TIPS
  • Column div(s) must be wrapped in a row div.
  • A single row can contain one column or many columns, but the number of columns should always add up to exactly twelve (3+9 / 4+8 / 12).
  • If you want equal column width for all devices larger than mobile, while keeping mobile at the default 100% width, you only need to specify the medium device class .col-m- (refer to the fluid grid and code example).
  • If you want equal column width on all devices, you only need to specify the small device class .col-.
  • Mix and match different device classes to get the perfect responsive design for your project.

Primary Buttons

The default primary <button> has an indigo background-color, white font color, and a subtle border-radius.

  
<div>
  <button>primary</button>
</div>

Use primary buttons to clearly and prominently indicate a call-to-action.

Secondary Buttons

Secondary buttons are an inverted version of primary buttons, and are best reserved for a call-to-action of a lower priority.

To use a secondary button, just add the class .color-light to a <button>.

View all color classes

  
<div>
  <button class="indigo-light">secondary</button>
</div>

When used alongside primary buttons, they are helpful for drawing the user's attention to a more preferred or recommended action.

Input Buttons

Use input buttons in HTML forms or for activating javascripts. The default <input type="submit">, <input type="reset">, and <input type="button"> all have an indigo background-color, white font color, and a subtle border-radius.

  
<div>
  <input type="submit" value="submit">
  <input type="reset" value="reset">
  <input type="button" value="button">
</div>

The .color-light class can be used with any <input> button to create a secondary <input> button.

  
<div>
  <input type="submit" value="submit" class="indigo-light">
  <input type="reset" value="reset" class="indigo-light">
  <input type="button" value="button" class="indigo-light">
</div>

Anchor Buttons

Anchor buttons are anchors that have <button> styling applied to them, and a simple arrow animation on hover.

To use an anchor button, add the class .btn-link to an anchor.

Note that for the hover animation to work, you must wrap the text of the anchor button within a <span>.

Anchor 
  
<div>
  <a href="#0" class="btn-link"><span>anchor </span></a>
</div>
The .btn-link class can also be used with the .color-light class to create a secondary anchor button.
anchor 
  
<div>
  <a href="#0" class="btn-link indigo-light"><span>anchor </span></a>
</div>

Button Color Classes

Use the .color class to change a button's background-color.

The .color class can be used with all indigo.css button types.


  
<div>
  <button class="red">red</button>
  <button class="orange">orange</button>
  <button class="gold">gold</button>
</div>

Colors can help convey different meanings and priority levels, depending on the context.

Use the .color-light class for a secondary button color scheme.

The .color-light class can be used with all indigo.css button types.


indigo purple pink grey black 
  
<div>
  <button class="red-light">red</button>
  <button class="orange-light">orange</button>
  <button class="gold-light">gold</button>

  ...

  <a href="#0" class="btn-link indigo-light"><span>indigo</span></a>
  <a href="#0" class="btn-link purple-light"><span>purple</span></a>
  <a href="#0" class="btn-link pink-light"><span>pink</span></a>
</div>

Use the .ghost class for a transparent background-color, white font color, and white border-color.

Ghost buttons are most effective when paired with high-contrast background colors or images that won't clash with the button's text.

The .ghost class can be used with all indigo.css button types.

  
<div>
  <button class="ghost">these</button>
  <button class="ghost">are</button>
  <button class="ghost">ghost</button>
  <button class="ghost">buttons</button>
</div>

Button Size Classes

Most indigo.css buttons are available in 6 sizes.

  1. Micro: .micro-btn
  2. Tiny: .tiny-btn
  3. Small: .small-btn
  4. Medium: <button>, <input>
  5. Large: .lrg-btn
  6. Huge: .huge-btn

Anchor buttons are currently available in 3 sizes.

  1. Medium: <a class="btn-link">
  2. Large: .lrg-btn
  3. Huge: .huge-btn
  
<div>
  <button class="micro-btn">micro</button>
  <button class="tiny-btn">tiny</button>
  <button class="small-btn">small</button>
  <button>medium</button>
  <button class="lrg-btn">large</button>
  <button class="huge-btn">huge</button>
</div>

Button Style Classes

Default indigo.css buttons have slightly rounded corners. Use the .square class or .pill class to change a button's border-radius for either a square-shaped button or a pill-shaped button.

Both of these classes can be used with all indigo.css button types.

  
<div>
  <button class="blue-light micro-btn square">square</button>
  <button class="blue tiny-btn square">square</button>
  <button class="blue-light small-btn square">square</button>
</div>
  

  
<div>
  <button class="purple micro-btn pill">pill</button>
  <button class="purple-light tiny-btn pill">pill</button>
  <button class="purple small-btn pill">pill</button>
</div>

Stickys are a type of label. They're highly versatile components that are useful for adding additional context, highlighting important information, alerting users with a message or notification, and much more.

In general, stickys are better suited for textual information, and badges are better suited for numerical information.

The default <badge> has an indigo background-color, white font color, and a subtle border-radius.
